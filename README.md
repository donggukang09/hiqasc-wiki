# HIQASC 수리부서 업무 위키

**DEVELOPED BY DONGGU KANG**

---

## 📋 프로젝트 소개

HIQASC 수리부서를 위한 업무 위키 시스템입니다.
네스프레소 & 브레빌 커피머신 수리 정보를 팀원들과 쉽게 공유할 수 있습니다.

### 주요 기능

✅ **신규 증상 등록** - 새로운 수리 증상을 간편하게 등록
✅ **증상 검색** - 모델, 에러코드, 증상으로 빠른 검색
✅ **실시간 필터링** - 모델별, 상태별, 태그별 필터
✅ **수리 매뉴얼** - 모델별 수리 가이드
✅ **업무 가이드** - DOA, 초과근무, 보고서 등 업무 프로세스
✅ **계정 불필요** - URL만 알면 누구나 접속 가능
✅ **모바일 지원** - 반응형 디자인

## 🚀 빠른 시작

### 방법 1: 로컬 스토리지 모드 (즉시 사용)

1. 모든 파일을 다운로드
2. `index.html` 파일을 브라우저에서 열기
3. 바로 사용 가능!

⚠️ **주의**: 로컬 스토리지는 브라우저별로 저장되므로 다른 컴퓨터에서는 데이터가 공유되지 않습니다.

### 방법 2: Google Sheets 연동 (추천)

모든 팀원이 데이터를 공유하려면 Google Sheets 연동이 필요합니다.

자세한 설정 방법은 **SETUP_GUIDE.md** 파일을 참고하세요.

## 💾 데이터 저장 방식

### 현재 (로컬 스토리지)
- 브라우저에 임시 저장
- 빠른 테스트 가능
- 데이터 공유 불가

### 연동 후 (Google Sheets)
- 중앙 데이터베이스
- 모든 팀원 실시간 공유
- 자동 백업
- Excel 내보내기 가능

## 🔧 사용 방법

### 1. 증상 등록하기

1. 메인 페이지에서 **🆕 신규 증상 등록** 클릭
2. 폼 작성:
   - 모델 선택 (ORIGINAL/VERTUO/BREVILLE)
   - 기종명 입력
   - 증상 설명
   - 해결 방법
   - 태그 선택 (Ctrl+클릭으로 여러 개)
3. **등록하기** 클릭

### 2. 증상 검색하기

1. 메인 페이지에서 **🔍 증상 검색** 클릭
2. 검색어 입력 또는 필터 선택
3. 결과 카드 클릭하여 상세 정보 확인

### 3. 매뉴얼 참고하기

1. **📋 수리 매뉴얼** 클릭
2. 모델별 가이드 확인

### 4. 업무 시스템 확인하기

1. **⚙️ 업무 가이드** 클릭
2. 각 시스템 사용법 참고

## 🌐 배포 방법

### GitHub Pages (무료 호스팅)

1. GitHub 계정 생성
2. 새 Repository 생성
3. 모든 파일 업로드
4. Settings → Pages → Source: main branch
5. 생성된 URL 팀원들과 공유

예시 URL: `https://yourusername.github.io/hiqasc-wiki`

### 다른 호스팅

- Netlify (무료)
- Vercel (무료)
- 회사 내부 서버

## ❓ 문제 해결

### 데이터가 저장되지 않아요
- 로컬 스토리지 모드인지 확인
- Google Sheets 연동 설정 확인 (SETUP_GUIDE.md)

### 검색이 안 돼요
- 데이터가 등록되어 있는지 확인
- 필터 초기화 버튼 클릭

### Google Sheets 오류
- Apps Script 배포 권한 확인
- 웹 앱 URL이 정확한지 확인
- 시트 이름이 "증상DB"인지 확인
